<Step title="Install">

<CodeGroup>

```bash npm
npm install greatstorage
```

```bash yarn
yarn add greatstorage
```

```bash pnpm
pnpm add greatstorage
```

</CodeGroup>

</Step>

<Step title="Create a storage instance">

Create a singleton so you share one configured instance across your app.

```typescript lib/storage.ts
import { createStorage } from 'greatstorage';

export const storage = createStorage({
  prefix: 'myapp',
});
```

The `prefix` option namespaces every key as `myapp:<key>`, so your entries never collide with other scripts writing to the same `localStorage`.

<Tip>
  Keep the storage singleton in a dedicated module (e.g. `lib/storage.ts`) and import it wherever you need it. Avoid calling `createStorage` multiple times with the same prefix.
</Tip>

</Step>

<Step title="Store and retrieve data">

```typescript
import { storage } from './lib/storage';

// Store a value
storage.setItem('username', 'alice');

// Retrieve it
const username = storage.getItem<string>('username');
console.log(username); // 'alice'

// Returns null if the key does not exist
const missing = storage.getItem<string>('does-not-exist');
console.log(missing); // null
```

`setItem` serializes the value and wraps it in greatstorage's internal envelope. `getItem` unwraps and deserializes on read, returning `null` for missing or expired entries.

</Step>

<Step title="Store rich types">

greatstorage uses [devalue](https://github.com/Rich-Harris/devalue) by default, so `Set`, `Map`, `Date`, `RegExp`, and `BigInt` round-trip correctly without any extra work.

```typescript
import { storage } from './lib/storage';

// Set
storage.setItem('tags', new Set(['typescript', 'browser', 'storage']));
const tags = storage.getItem<Set<string>>('tags');
console.log(tags instanceof Set); // true
console.log(tags?.has('typescript')); // true

// Map
storage.setItem('scores', new Map([['alice', 42], ['bob', 37]]));
const scores = storage.getItem<Map<string, number>>('scores');
console.log(scores instanceof Map); // true
console.log(scores?.get('alice')); // 42

// Date
storage.setItem('lastSeen', new Date());
const lastSeen = storage.getItem<Date>('lastSeen');
console.log(lastSeen instanceof Date); // true
```

</Step>

<Step title="Add expiration">

Attach a `ttl` (milliseconds from now) or an exact `expiresAt` timestamp to any entry.

```typescript
import { storage } from './lib/storage';

// Expire after 60 seconds
storage.setItem('sessionToken', 'abc123', { ttl: 60_000 });

// Expire at a specific time
storage.setItem('promoCode', 'SAVE20', {
  expiresAt: new Date('2026-12-31T23:59:59Z'),
});

// Expired entries return null automatically
const token = storage.getItem<string>('sessionToken');
```

When you read an expired entry, greatstorage removes it from storage and returns `null`. You can also sweep all expired entries at once:

```typescript
storage.clearExpired();
```

</Step>

<Step title="Add schema validation">

Pass a schema with a `parse` method to validate data on read. [Zod](https://zod.dev) is a natural fit.

```typescript
import { z } from 'zod';
import { storage } from './lib/storage';

const UserSchema = z.object({
  id: z.string(),
  name: z.string(),
  role: z.enum(['admin', 'member']),
});

type User = z.infer<typeof UserSchema>;

// Store a user object
storage.setItem<User>('currentUser', {
  id: 'usr_1',
  name: 'Alice',
  role: 'admin',
});

// Retrieve with validation — returns null if schema fails
const user = storage.getItem<User>('currentUser', { schema: UserSchema });
console.log(user?.name); // 'Alice'
```

If the stored value does not match the schema (for example, because a previous version of your app stored a different shape), `getItem` returns `null` instead of throwing.

</Step>
